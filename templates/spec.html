仕様書

RECO3の動作仕様と機能一覧

デプロイメントモード

PCエージェントは任意です。必要に応じて選択してください。

モード A: PWA のみ

推奨用途：LLM出力品質の監視のみが必要な場合

  • セットアップ: SECRET_KEY + LLM API鍵のみ
  • オンライン中に監視・制御ループが動作
  • PC側のプロセス監視/制御は不可
  • 音なし画面通知（PWA開いている時）
  • コスト: 最小限

モード B: PWA + PCエージェント

推奨用途：PC制御を行いたい場合

  • PC側のプロセス監視・制御・自動再起動
  • フェイルセーフ自動化（SAFE MODE）
  • リアルタイム CPU/Memory/ディスク監視
  • エージェントのオンライン/オフライン状態表示
  • Windows/macOS サービス自動起動

コア機能

Input Gate

入力テキスト・音声を分析

  • 曖昧度スコア
  • 感情圧力検出
  • 非現実性判定
  • 0.0～1.0 の数値化

Output Gate

LLM回答を自動検査

  • 断定密度スコア
  • エビデンス不足検出
  • 内部矛盾検査
  • 品質スコア自動付与

自動制御ループ

品質に基づく自動制御

  • スコア <閾値: 自動再生成
  • 60秒クールダウン
  • 連続失敗3回で自動停止
  • アラート通知

監査ログ

全セッション記録

  • 入出力スコア
  • 判定結果（verdict）
  • ユーザーフィードバック
  • API取得可能

API エンドポイント

エンドポイント 説明
POST /api/r3/chat チャット / 分析リクエスト
POST /api/r3/analyze_input 入力のみ分析
POST /api/r3/analyze_output 出力のみ分析
GET /api/r3/config 設定取得
GET /api/status システムステータス（LLM設定等）
GET /api/logs 監査ログ取得
GET /agent/pull エージェントコマンド取得（モードB）
POST /agent/heartbeat エージェントハートビート（モードB）
POST /agent/logs エージェントログ送信（モードB）

セキュリティ

認証

  • PWA: サーバーサイドセッション
  • API: X-API-Key ヘッダー
  • エージェント: X-AGENT-ID + X-API-Key

データ保護

  • API鍵: サーバー側のみ（フロント非埋込）
  • Cookie: HttpOnly セッション
  • CORS: 同一オリジン
